# 多语言文档

`iztro-rs` 从一开始就需要多语言文档。至少必须支持英文和中文。

## 规范来源

- 英文文档是工程规格的规范来源。
- 中文文档是紫微斗数术语的规范来源。
- 若行为和术语出现冲突，行为以英文工程规格为准，术语应参考中文术语表。

## 必备文档

主要文档都应提供英文和中文版本：

- README；
- 项目规格；
- 架构；
- 路线图；
- 兼容性政策；
- 术语表；
- 规则引擎设计；
- ADR。

## 文件布局

```text
docs/
  en/
    ...
  zh-CN/
    ...
```

根目录文档可使用语言后缀，例如：

```text
README.md
README.zh-CN.md
CONTRIBUTING.md
CONTRIBUTING.zh-CN.md
```

## 实现说明

代码不应假设中文字符串是内部表示。内部逻辑应使用稳定 key 或 enum。多语言本地化应发生在输出边界。

## Review 政策

文档 PR 应说明是否同步更新了两种语言。如果只更新一种语言，应解释原因。
