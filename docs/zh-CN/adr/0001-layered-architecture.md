# ADR 0001：分层架构

## 状态

已接受。

## 背景

`iztro-rs` 需要支持确定性排盘、结构化特征提取、基于规则的解读，以及人类可读报告。如果这些职责混在一起，项目会很难测试和扩展。

## 决策

项目采用四个概念层：

1. Core Chart Layer。
2. Feature Extraction Layer。
3. Rule Engine Layer。
4. Narrative Layer。

每一层消费下一层输出，并避免把上层职责泄漏到下层。

## 影响

- 排盘逻辑可以独立于解读逻辑测试。
- 规则匹配可以独立于自然语言渲染测试。
- 叙事输出可以在不修改排盘算法的情况下变化。
- Coding agent 可以分别为各层搭建脚手架。
