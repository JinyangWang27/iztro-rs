# 项目规格

`iztro-rs` 是一个 Rust 版紫微斗数项目，目标是提供排盘、特征提取，以及未来基于规则的解盘能力。

## 产品边界

本项目是引擎和库，不是面向终端用户的算命产品。它应该提供确定性的数据结构和解释原语，供 CLI、Python binding、WebAssembly binding 或外部应用消费。

## 核心目标

1. 提供强类型的紫微斗数星盘模型。
2. 在适用范围内复现与 `iztro` 兼容的排盘行为。
3. 将排盘事实与解读逻辑分离。
4. 将解盘表示为带证据的结构化判断。
5. 支持中英文文档，并为未来多语言输出保留空间。
6. 为多派别、方法 profile、解读风格保留扩展能力。

## 早期范围

第一阶段应覆盖：

- Rust workspace 脚手架；
- 核心领域枚举和结构体；
- 星盘数据模型序列化；
- 特征提取 trait 和占位实现；
- 规则 schema 和规则匹配骨架；
- 确定性报告数据结构；
- 后续与 `iztro` 对比的 golden test 基础设施设计。

## 早期非范围

- 完整紫微斗数规则知识库。
- 完整飞星四化或钦天体系。
- 自动经验权重学习。
- 以 LLM 直接解盘为核心。
- 生产级自然语言解盘。
- 完整兼容 `iztro` 的所有公开方法。

## 设计原则

- 先确定性，后表达力。
- 先强类型，后字符串规则逻辑。
- 先结构化判断，后自然语言。
- 解读必须证据优先。
- 用 strategy/profile 组合支持多派别。
- 英文工程规格与中文术语表共同维护。
